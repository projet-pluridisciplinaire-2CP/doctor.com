# Plan: Tighten AI System Prompt + Update Test Document

## Context

The AI Q&A module is functionally complete. However, Q1 testing ("Quels sont les antecedents de ce patient?") revealed the AI produces overly verbose responses that go outside the scope of the question. Specifically:
- Included habitudes de vie, voyages, traitements, vaccinations — none of which are "antecedents"
- Added a speculative "Donnees manquantes" section about missing data that wasn't asked for
- Response was far too long for a simple factual retrieval question

## Task 1: Tighten the system prompt

### File: `packages/api/src/modules/ai-q&a/service.ts` (line 54-55)

Replace the current system prompt string with a stricter version that enforces:

1. **Answer ONLY the question asked** — no tangential information from other data sections
2. **Stay concise** — short focused answers, not exhaustive data dumps
3. **No speculation** — don't mention what's missing unless the user's specific question can't be answered
4. **Keep markdown formatting** — but don't over-structure simple answers

**Current prompt (line 54-55):**
```
"Vous etes un assistant medical destine aux medecins. Le medecin vous pose une question a propos d'un patient. Repondez UNIQUEMENT en vous basant sur les donnees du patient fournies ci-dessous. Si l'information demandee n'est pas disponible dans les donnees, indiquez-le clairement. Repondez de maniere concise, factuelle et en francais. Formatez votre reponse en Markdown (titres, listes, gras, etc.) pour une lecture claire et structuree."
```

**New prompt:**
```
"Vous etes un assistant medical destine aux medecins. Regles strictes :\n\n1. Repondez UNIQUEMENT a la question posee — ne mentionnez pas d'informations provenant d'autres sections des donnees qui ne sont pas directement liees a la question.\n2. Basez-vous exclusivement sur les donnees du patient fournies ci-dessous.\n3. Soyez bref et direct : quelques lignes suffisent pour une question factuelle simple. N'ajoutez pas de recapitulatif, de contexte supplementaire ou de sections non demandees.\n4. Ne speculez pas sur les donnees manquantes. Mentionnez l'absence d'information UNIQUEMENT si la question porte specifiquement sur une donnee absente.\n5. Repondez en francais. Utilisez le Markdown (listes, gras) uniquement si cela ameliore la lisibilite — pas de titres inutiles pour les reponses courtes."
```

Key changes:
- Explicit rule about not pulling in unrelated data sections
- "Quelques lignes suffisent" for factual questions
- No speculation about missing data unless the question specifically can't be answered
- Markdown should be proportional to answer complexity (no overformatting)
